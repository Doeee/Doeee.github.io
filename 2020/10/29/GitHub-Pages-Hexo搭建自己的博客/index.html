GitHub Pages+Hexo搭建博客

①首先需要安装Node.js也需要配置本地的Git环境，这个下次再总结，直接从安装个人博客网站框架Hexo开始介绍

安装hexo指令

$ npm install hexo-cli -g

安装 hexo 部署到 git page 的 deployer

$ npm install hexo-deployer-git --save

②Hexo的初始化设置

新建一个Hexo文件夹，在文件下中右键点击Git Bash Here并初始化Hexo

$ hexo init

③现在即可查看网站了

Hexo生成静态文件

$ hexo generate

启动服务器

hexo server

输入http://localhost:4000/即可查看博客初始界面

④将博客部署到GitHub Page

这需要一个Git账户

域名必须是：Git名字.github.io

并且勾选Initialize this repository with a README

然后配置SSH密钥，这个部分下次再总结，本笔记主要是记录搭建Hexo博客的流程

⑤将本地的Hexo文件更新到GitHub库中

打开刚刚在github上建立的项目和在本地创建的Hexo文件夹中的_config.yml 文件进行配置（所有冒号后有一个空格）

然后输入$ hexo g -d命令发布（如果在之前修改过页面先用hexo clean && hexo deploy这个命令发布）

⑥发布一篇文章

在hexo文件夹下git bash输入

$ hexo n "文章标题"

会在项目 \Hexo\source_posts中生成 文章标题.md 文件，当然也可以直接在 \Hexo\source_posts 目录下右键鼠标新建文本文档，改后缀为 .md 即可，这种方法比较方便

⑦更换博客主题

https://hexo.io/themes/

这个网站是Hexo官网的主题专栏，有许多的主题供选择，在每个主题的最下面有Theme By XX主题 点击进去，在themes下git bash安装主题和所需要的插件，我选择的是Butterfly这个主题https://demo.jerryc.me/posts/21cfbf15/#%E5%AE%89%E8%A3%9D

安装主题：

git clone -b master https://github.com/jerryc127/hexo-theme-butterfly.git themes/butterfly

安装插件：

npm install hexo-renderer-pug hexo-renderer-stylus --save

然后发现themes下有个themes文件夹下有我刚刚下好的butterfly，把这个文件夹移到外面的themes和landscape同级

然后打开Hexo 文件夹下的配置文件 _config.yml ，找到关键字 theme，修改参数（注意冒号后面有一个空格）

然后返回Hexo目录,右键 Git Bash Here ，输入以下命令开始部署主题

$ hexo g

$ hexo s

此时访问 http://localhost:4000/ 可以看见主题已更换

然后继续输入

$ hexo clean //该命令的作用是清除缓存，若不输入此命令，服务器有可能更新不了主题

$ hexo g -d

⑧关于hexo上无法显示.md图片问题

方法有下面三个，我用的第二个比较方便

  1. 绝对引用: 使用 /source/images 文件夹
  2. 相对引用: 使用 post_asset_folder:true 建立文章同名文件夹
  3. CDN 或者图床引用

比如我要引入图片aaa.png:![Lena](GitHub-Pages-Hexo搭建自己的博客/aaa.png)

⑨记录自己遇到的奇怪的问题

第⑦步更换博客主题的最后 hexo g -d的时候我发现我的分支在master而我的默认分支是main，所以我的博客没有更新 然后我把我的默认分支改成了master依然不更新，直到我改成master下的doc才更新

但是更改以后又出了问题，我无法hexo g -d更新我的git仓库，然后我只有把文件改成master下的root才能更新仓库

我也不知道为啥会这样，问题暂时解决了，但是却不知道原因，如果有大佬知道可以联系我的qq1127192571交流一下~
